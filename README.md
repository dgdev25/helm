# Deathstar — Project Dashboard

Centralized dashboard for tracking all your GitHub repos and local git projects.

## Features

- **GitHub sync**: Automatically fetches repositories, issues, and recent commits from GitHub via Octokit
- **Local scanning**: Discovers and tracks local git projects from configurable directories
- **Dashboard**: Dark-themed React interface with real-time project data, language filtering, and search
- **Scheduled sync**: Automatic synchronization on configurable intervals (default: every 6 hours)
- **Manual refresh**: Click "Sync Now" in the sidebar or POST to `/api/sync`
- **Project status**: Track active, paused, and archived projects
- **Recent activity**: View last commit times, messages, and authors

## Setup

### Prerequisites

- Node.js 18+
- PostgreSQL (local or Docker)
- GitHub token (for API access)

### Installation

1. **Clone and install:**
   ```bash
   npm install
   ```

2. **Configure environment:**
   ```bash
   cp .env.example .env
   ```
   
   Fill in required variables:
   - `DATABASE_URL` - PostgreSQL connection string (e.g., `postgres://user:pass@localhost:5433/deathstar`)
   - `GITHUB_TOKEN` - Personal access token from GitHub (scopes: `repo`, `read:user`)
   - `GITHUB_USERNAMES` - Comma-separated list of GitHub usernames to sync
   - `LOCAL_SCAN_DIRS` - Comma-separated paths to scan for local git projects
   - `SYNC_INTERVAL_HOURS` - Sync frequency in hours (default: 6)
   - `PORT` - Server port (default: 7337)

3. **Initialize database:**
   ```bash
   psql $DATABASE_URL -f server/schema.sql
   ```

## Development

Development runs both the Fastify API server and Vite dev server concurrently:

```bash
npm run dev
```

- **API server:** http://localhost:7337 (non-standard port — note this!)
- **Vite dev server:** http://localhost:7338 (non-standard port — note this!)
- **Frontend:** Open http://localhost:7338 (proxies `/api` to port 7337)

The Vite dev server automatically hot-reloads changes and proxies API requests to the backend.

### Development Scripts

- `npm run dev:server` - Run Fastify server with file watching
- `npm run dev:client` - Run Vite dev server only
- `npm run build` - Build frontend for production

## Production

Build and serve the app in production mode:

```bash
npm run build
npm start
```

The production server:
- Serves pre-built static assets from `dist/`
- Listens on **port 7337** (non-standard port)
- Requires `NODE_ENV=production`
- Disables CORS (client and server on same origin)

Visit http://localhost:7337 after starting.

## Synchronization

### Automatic Sync

The scheduler starts automatically and syncs every `SYNC_INTERVAL_HOURS` (default: 6 hours):
- Fetches GitHub repositories and their recent commits
- Scans local directories for new git projects
- Updates the database with latest metadata

### Manual Sync

**Via API:**
```bash
curl -X POST http://localhost:7337/api/sync
```

**Via UI:**
Click "Sync Now" in the dashboard sidebar.

### Sync Logs

View recent sync history:
```bash
curl http://localhost:7337/api/sync/log
```

## API Endpoints

- `GET /api/health` - Health check
- `GET /api/projects` - List all projects (query: `search`, `status`, `language`)
- `GET /api/projects/:slug` - Get project details
- `PATCH /api/projects/:slug` - Update project (status, description)
- `POST /api/projects/:slug/sync` - Sync single project
- `POST /api/sync` - Trigger full sync
- `POST /api/scan/local` - Scan local directories
- `GET /api/sync/log` - View sync history

## Database

PostgreSQL schema includes:
- `projects` - Repository metadata
- `github_sync_log` - Sync history and status

See `server/schema.sql` for full schema.

## Troubleshooting

**Port already in use:**
```bash
# Kill process on port 7337
lsof -ti:7337 | xargs kill -9

# Or use a different port
PORT=8000 npm start
```

**GitHub API rate limits:**
Ensure your token has sufficient permissions and check GitHub's rate limit status in sync logs.

**Database connection errors:**
Verify `DATABASE_URL` is correct and PostgreSQL is running:
```bash
psql $DATABASE_URL -c "SELECT 1"
```
